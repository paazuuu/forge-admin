# 应用概览页布局重构

## 现状分析

当前 `app-center/index.vue` 页面存在以下问题：
1. **扁平化展示** — 所有业务对象（主对象/明细对象/查找对象/单据）平铺为卡片网格，无法体现主从归属关系
2. **按钮过多** — 每个卡片底部3个按钮 + 右上角菜单 + 入口行菜单，操作分散
3. **类型不明显** — objectType（MASTER/DETAIL/LOOKUP/TRANSACTION）只用一个小标签显示，用户很难快速识别
4. **无功能聚合** — "采购单"(主对象) 和 "采购明细"(明细对象) 虽然有关系但各自独立展示

## 设计目标

参考飞书多维表格"关联表视图" + 简道云"应用管理" + Airtable "Interface Designer"：
- 主对象作为一级分组容器，关联的明细/子表对象嵌套在其中
- 卡片只保留1个主操作按钮，其余收进 `...` 菜单
- 类型通过视觉层级和结构关系自然表达，而非依赖标签

## 重构方案

### Task 1: 数据层 — 构建主从分组树

**文件**: `forge-admin-ui/src/views/app-center/index.vue`

在现有 `businessUnits` computed 之后，新增一个 `groupedBusinessUnits` computed，按主从关系重组数据：

```
逻辑：
1. 从 objects 列表中识别所有 MASTER/TRANSACTION 类型对象作为"主对象"
2. 调用 API 获取关系列表（可复用已有的 businessObjectRelations），找到每个主对象关联的 DETAIL 子对象
3. 构建树形结构：主对象包含其 detailObjects 数组
4. 没有被任何主对象引用的 DETAIL/LOOKUP 对象归入"未归属对象"组
5. 排序：按 sortOrder -> objectName 排序
```

由于现有 API `businessObjectList` 返回的数据已包含 `objectType` 和 `relationCount`，但缺少具体关系信息。需要：
- **方案A（推荐）**：新增后端接口 `GET /ai/business/object/list-with-relations`，一次性返回对象列表 + 每个对象的关系摘要（targetObjectCode + relationType）
- **方案B（纯前端）**：对有 `relationCount > 0` 的对象，批量请求 `businessObjectRelations(objectId)` 获取关系

### Task 2: UI层 — 新增 BusinessObjectGroup 组件

**新文件**: `forge-admin-ui/src/views/app-center/components/BusinessObjectGroup.vue`

功能：替代现有的 BusinessUnitCard，展示"主对象 + 关联对象"分组：

```
布局结构：
┌─────────────────────────────────────────────────────┐
│ [图标] 采购管理                           [设计] [...] │  ← 主对象头部
│ 业务单据 · 3个关系 · 2个入口                          │
├─────────────────────────────────────────────────────┤
│  ├── 📋 采购单（主表）      [打开] [...]              │  ← 主对象自身
│  ├── 📑 采购明细（明细）    [打开] [...]              │  ← 关联明细对象
│  └── 📑 供应商（引用）      [打开] [...]              │  ← 关联查找对象
├─────────────────────────────────────────────────────┤
│  访问入口：                                          │
│    ● 采购列表页（运行态）  ● 采购录入（表单）          │  ← 入口列表
└─────────────────────────────────────────────────────┘
```

设计要点：
- 一级卡片 = 主对象所属的功能组（用 objectName 或 suiteName 作为组名）
- 对象列表用缩进树形展示，每行只有"打开"主按钮 + `...` 省略号菜单
- 访问入口折叠在底部，默认收起，点击展开
- objectType 用图标/颜色区分（MASTER=蓝色实心方块，DETAIL=灰色缩进，LOOKUP=虚线边框，TRANSACTION=橙色标记）

### Task 3: UI层 — 操作按钮收敛

对于每个对象行：
- **主操作**：只保留"设计"按钮（点击进入 object-designer）
- **省略号菜单**：编辑、统计看板、启用/停用、删除、业务流程配置、添加入口

对于每个访问入口：
- **主操作**：只保留"打开"按钮
- **省略号菜单**：配置、下载代码、启用/停用、删除

### Task 4: 页面顶部 — 简化 metric 和工具栏

- metric 区域从4个指标精简为3个：业务对象、访问入口、已就绪（可直接打开）
- 筛选栏增加 objectType 筛选（主对象/明细/查找/单据），方便快速定位
- "新建"下拉改为更明确的两个选项：新建业务对象 / 新建访问入口

### Task 5: 样式 — 树形分组卡片视觉

- 主对象组用左侧色条标记类型（MASTER=蓝，TRANSACTION=橙）
- 子对象行缩进 24px，用连接线表示从属关系
- hover 时显示操作区域，非 hover 时只显示对象名+类型图标+状态
- 卡片间距减小，紧凑排列，一屏展示更多信息

### Task 6: 后端接口（可选）— 对象关系摘要

如果选择方案A（推荐），需要后端新增：

**文件**: `forge-server/forge-framework/forge-plugin-parent/forge-plugin-generator/src/main/java/.../controller/BusinessObjectController.java`

新增接口 `GET /ai/business/object/list-with-relations`，在返回对象列表时附带关系摘要字段：
```java
// BusinessObjectVO 增加字段
private List<RelationSummary> relations; // [{targetObjectCode, targetObjectName, relationType}]
```

## 交互体验改进

| 场景 | 现状 | 改进后 |
|------|------|--------|
| 找到采购明细 | 在扁平网格中逐一扫描 | 在"采购管理"组内直接看到树形子项 |
| 操作某个对象 | 3个底部按钮+右上角菜单，选择困难 | 1个主按钮"设计"+`...`收纳其余 |
| 区分类型 | 只有一个小标签 | 通过层级关系+图标+色条自然表达 |
| 新建关联对象 | 需要先新建对象再手动配关系 | 在主对象组内"添加子对象"直接创建带关系 |

## 实施优先级

1. Task 2 + Task 3（UI组件+操作收敛）— 用户体验提升最明显
2. Task 1（数据分组逻辑）— 结合现有数据构建前端分组
3. Task 4 + Task 5（metric/样式优化）— 打磨细节
4. Task 6（后端接口）— 性能优化，可后置

## 需确认的问题

开始实施前需要确认：
1. 对象关系数据获取方式 — 方案A（新增后端接口，性能好但需改后端）vs 方案B（前端批量请求，不需改后端但有N+1问题）？
2. 是否需要保留现有的卡片网格视图作为可切换的备选视图模式？
