# 表单与列表设计器配置去重优化

## 背景

当前低代码对象设计器中，**弹窗方式、弹窗宽度、编辑表单列数/间距、标签位置/对齐** 等配置在 3 个地方重复出现：

| 配置项 | 位置 1：表单属性→表单项配置 | 位置 2：CRUD 选项→编辑弹窗 | 位置 3：列表设计→表单与弹窗 |
|--------|---------------------------|--------------------------|--------------------------|
| 编辑打开方式 | `ForgePropertyPanel` L3286 | `ForgePropertyPanel` L2890 | `ListPageGridDesigner` L1785 |
| 弹窗宽度 | ❌ 无 | `ForgePropertyPanel` L2898 | `ListPageGridDesigner` L1797 |
| 详情弹窗宽度 | ❌ 无 | `ForgePropertyPanel` L2905 | `ListPageGridDesigner` L1806 |
| 编辑表单列数/间距 | `ForgePropertyPanel` L3294（gridColumns） | `ForgePropertyPanel` L2867-2888 | `ListPageGridDesigner` L1730-1782 |
| 标签位置/对齐/宽度 | `ForgePropertyPanel` L3343-3392 | ❌ 无（走 editLabelWidth） | `ListPageGridDesigner` L1748-1756 |

此外，**字段最大长度** 不够突出，**组件类型切换** 在表单设计器中缺失。

## 目标

1. **单一配置源**：弹窗方式/宽度/详情宽度/编辑列数/间距 统一在 `ForgePropertyPanel` 的"表单属性→表单项配置"中配置
2. **消除重复**：CRUD 选项 Drawer 和 ListPageGridDesigner 中的重复编辑项改为只读摘要 + 跳转提示
3. **表单设计器添加组件类型切换**：选中字段组件后可切换 componentKey

## Task 5：ForgePropertyPanel 增加组件类型切换

**文件**：`forge-admin-ui/src/views/app-center/components/designer/forge-form-designer/ForgePropertyPanel.vue`

**位置**：选中组件属性面板 → "标识" collapse-item（L41-53）内，`label` 输入框下方

**新增 UI**：

```vue
<n-form-item v-if="canSwitchComponentType" label="组件类型">
  <n-select
    :value="selectedComponent.componentKey"
    :options="switchableComponentOptions"
    filterable
    @update:value="handleSwitchComponentType"
  />
</n-form-item>
```

**新增逻辑**（script 区）：

```js
// 可切换的组件类型映射（同类组件之间可互换）
const SWITCHABLE_GROUPS = [
  ['input', 'textarea', 'number', 'inputNumber', 'money'],
  ['select', 'radio', 'checkbox', 'dictSelect'],
  ['date', 'datetime', 'dateRange'],
  ['fileUpload', 'imageUpload'],
  ['objectReference', 'recordSelector'],
]

const canSwitchComponentType = computed(() => {
  if (!selectedComponent.value) return false
  const key = selectedComponent.value.componentKey
  return SWITCHABLE_GROUPS.some(group => group.includes(key))
})

const switchableComponentOptions = computed(() => {
  const key = selectedComponent.value?.componentKey
  const group = SWITCHABLE_GROUPS.find(g => g.includes(key)) || []
  return group.map(k => ({
    label: COMPONENT_LABEL_MAP[k] || k,
    value: k,
  }))
})

const COMPONENT_LABEL_MAP = {
  input: '单行输入',
  textarea: '多行文本',
  number: '数字输入',
  inputNumber: '数字输入',
  money: '金额',
  select: '下拉选择',
  radio: '单选框',
  checkbox: '多选框',
  dictSelect: '字典选择',
  date: '日期',
  datetime: '日期时间',
  dateRange: '日期范围',
  fileUpload: '文件上传',
  imageUpload: '图片上传',
  objectReference: '引用对象',
  recordSelector: '记录选择器',
}

function handleSwitchComponentType(newKey) {
  if (!selectedComponent.value || newKey === selectedComponent.value.componentKey) return
  const oldKey = selectedComponent.value.componentKey
  // 保留现有 props 中通用属性，清理特定组件属性
  const commonProps = ['defaultValue', 'placeholder', 'disabled', 'clearable', 'required']
  const preserved = {}
  commonProps.forEach(prop => {
    if (selectedComponent.value.props?.[prop] !== undefined) {
      preserved[prop] = selectedComponent.value.props[prop]
    }
  })
  updateComponent({
    componentKey: newKey,
    props: { ...preserved },
  })
  // 同步更新字段绑定的 fieldType/dataType（基于 FIELD_TYPE_DEFAULTS 映射）
  const defaults = FIELD_TYPE_DEFAULTS[newKey]
  if (defaults && selectedComponent.value.fieldBinding) {
    updateFieldBinding({
      fieldType: defaults.fieldType,
      dataType: defaults.dataType,
      componentType: defaults.componentType || newKey,
    })
  }
}
```

**注意**：`FIELD_TYPE_DEFAULTS` 已存在于 L4844 附近（即 `input: { fieldType: 'TEXT', ... }` 映射表），直接复用。
