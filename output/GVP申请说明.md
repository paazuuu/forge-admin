# GVP 申请说明 — Forge Admin

---

## 一、项目简介

**Forge Admin** 是一套基于 **Vue 3 + Spring Boot 3** 的企业级中后台管理框架，采用**微内核插件化**架构设计，内置 **AI 智能体引擎**、**AI 数据可视化大屏**、**低代码表单/列表设计器**、**Flowable 工作流**、**多租户隔离**等核心能力，为企业后台、SaaS 管理端、数据可视化平台和内部低代码工具提供一站式解决方案。

与传统后台框架不同，Forge Admin 不仅提供用户、角色、菜单、字典等基础能力，更将 **AI 赋能开发** 作为核心设计目标——通过内置的 AI 智能体引擎，开发者可以用自然语言生成业务系统、流程模型、数据大屏和 CRUD 页面，大幅提升开发效率。

- **项目地址**：https://gitee.com/ForgeLab/forge-admin
- **在线演示**：http://www.dlforgelab.com:8084/forge/login
- **官方文档**：http://www.dlforgelab.com:8084/forge-docs/
- **开源协议**：Apache 2.0

## 二、项目背景与意义

### 2.1 行业痛点

当前企业级后台开发面临三大痛点：

1. **重复劳动多**：每个项目都要从零搭建用户、角色、权限、日志等基础模块，开发效率低下
2. **技术选型杂**：认证、缓存、ORM、工作流、文件存储等技术栈组合复杂，团队学习成本高
3. **AI 落地难**：大模型时代，开发者希望用 AI 提升效率，但缺乏工程化的落地手段

市面上现有的后台框架要么功能单薄（只有基础 CRUD），要么架构臃肿（难以扩展），要么缺乏 AI 能力（停留在传统开发模式）。

### 2.2 项目价值

Forge Admin 旨在打造一套**可持续演进的企业级后台工程体系**：

- **提效**：通过低代码设计器和 AI 代码生成，减少 80% 的重复编码
- **稳定**：插件化架构保证核心稳定，业务能力按需组装
- **前瞻**：内置 AI 智能体引擎，拥抱大模型时代的开发范式变革

## 三、核心功能与技术亮点

### 3.1 微内核插件化架构

Forge Admin 采用 **微内核 + 插件化** 的分层设计，是项目最核心的技术创新：

```
应用层 (Application)
    ↓
业务插件层 (Plugin) — 8 大业务插件
    ↓
技术启动层 (Starter) — 20 个技术 Starter
    ↓
核心框架层 (Framework)
```

**技术启动层（20 个 Starter）**：
- 认证授权（forge-starter-auth）
- 缓存抽象（forge-starter-cache）
- ORM 封装（forge-starter-orm）
- 多租户隔离（forge-starter-tenant）
- 数据权限（forge-starter-datascope）
- 字段加解密（forge-starter-crypto）
- 操作日志（forge-starter-log）
- 文件存储（forge-starter-file）
- Excel 导入导出（forge-starter-excel）
- 任务调度（forge-starter-job）
- 消息中心（forge-starter-message）
- 接口幂等（forge-starter-idempotent）
- 第三方登录（forge-starter-social）
- WebSocket（forge-starter-websocket）
- 分布式事务（forge-starter-trans）
- ...

**业务插件层（8 大插件）**：
- 系统管理插件（forge-plugin-system）— 用户、角色、菜单、字典、组织
- 代码生成插件（forge-plugin-generator）— 低代码设计器、代码生成
- AI 智能体插件（forge-plugin-ai）— AI 智能体引擎、模型管理
- 工作流插件（forge-plugin-flow）— Flowable 集成、流程设计
- 任务调度插件（forge-plugin-job）— 定时任务管理
- 消息中心插件（forge-plugin-message）— 站内信、通知模板
- 数据可视化插件（forge-plugin-data）— 大屏设计、报表
- 外部集成插件（forge-plugin-external）— 第三方系统对接

**架构优势**：
- ✅ **按需加载**：用什么插件引什么，避免臃肿
- ✅ **独立演进**：插件之间解耦，可单独升级
- ✅ **易于扩展**：新增能力只需开发新插件，不影响核心
- ✅ **企业级**：每个 Starter 都经过生产环境验证

### 3.2 AI 智能体引擎（核心差异化能力）

Forge Admin 内置 **AI 智能体引擎**，是国内少有的将 AI Agent 工程化落地的开源后台框架。

**内置 7 大智能体**：

| 智能体 | 功能描述 |
|--------|----------|
| 🧠 低代码业务系统生成 Agent | 根据业务需求自动划分领域、生成数据模型和低代码应用草稿 |
| 📋 流程 BPMN 生成助手 | 自然语言描述 → 自动生成 Flowable BPMN XML 流程配置 |
| 🗄️ CRUD 配置生成器 | 自然语言描述或数据库表结构 → CRUD 配置 JSON |
| 📊 大屏生成助手 | 根据用户需求自动生成数据可视化大屏布局 |
| 💬 智能客服 | 内置 AI 对话能力，可接入业务知识库 |
| 🔧 代码生成字段顾问 | 根据数据库字段信息推荐 Java 类型、表单组件、字典类型、验证规则 |
| 🏗️ 代码生成 Schema 构建器 | 根据自然语言描述推断数据模型 Schema |

**智能体引擎特性**：
- 完整的智能体生命周期管理（草稿 → 发布 → 下线）
- 支持配置模型、温度、上下文等参数
- 支持 MCP 工具扩展
- 每个智能体独立管理会话和提示词模板
- 会话记录持久化，支持回溯和复盘

**多 AI 供应商支持**：
- 阿里百炼（通义千问）
- OpenAI（GPT 系列）
- DeepSeek
- Ollama（本地部署）
- 智谱 AI（GLM 系列）
- Moonshot（月之暗面）
- 可扩展接入更多供应商

### 3.3 AI 数据可视化大屏

**Forge AI 大屏** 是项目的明星功能，支持**自然语言生成可视化大屏**：

- **AI 生成**：用自然语言描述需求，AI 自动生成大屏布局和组件
- **拖拽编辑**：可视化编辑器，支持组件拖拽、调整大小、配置属性
- **主题定制**：内置多套主题，支持自定义配色方案
- **真实数据**：支持接入真实 API 数据源，实时展示业务数据
- **一键发布**：设计完成后一键发布为可访问页面
- **组件丰富**：折线图、柱状图、饼图、雷达图、地图、数字翻牌器等 30+ 组件

### 3.4 低代码表单/列表设计器

Forge Admin 的低代码设计器经过多次迭代优化，已达到生产可用级别：

**表单设计器**：
- 支持 30+ 表单组件（输入框、下拉、日期、上传、富文本、级联选择等）
- 可视化拖拽布局
- 组件属性配置（校验规则、默认值、占位符等）
- 联动逻辑配置（显示隐藏、禁用启用、计算赋值）
- 支持分组、标签页等复杂布局

**列表设计器**：
- 多级表头支持
- 行列合并配置
- 自定义列模板
- 排序、筛选、搜索配置
- 批量操作按钮配置
- 导出导入模板配置

**核心优势**：
- ✅ **生成生产级代码**：不是中间格式，直接生成可用的 Vue3 + Spring Boot 代码
- ✅ **双向同步**：设计器改了代码同步，代码改了设计器也能识别
- ✅ **可下载代码包**：生成的代码可以下载后二次开发
- ✅ **一键生成完整模块**：前端页面 + 后端接口 + 菜单权限 + 字典配置

### 3.5 企业级基础能力

**多租户 SaaS 能力**：

## 七、申请 GVP 的理由

### 7.1 技术创新性

Forge Admin 在多个方面具有技术创新：

1. **微内核插件化架构**：在开源后台框架领域，这种分层清晰的插件化架构较为少见，为企业级项目的可持续演进提供了优秀范例
2. **AI 智能体引擎**：将 AI Agent 工程化落地到后台框架中，是对"AI + 开发工具"融合的积极探索
3. **低代码 + 代码生成双向同步**：解决了传统低代码平台"生成的代码没法用"的痛点
4. **AI 数据大屏**：自然语言生成可视化大屏，降低了数据可视化的门槛

Forge Admin 不是玩具项目，而是真正能在生产环境使用的企业级框架：
