# 我们亲手把 Forge Admin 拆了，然后重装了一遍

> 一个做了两年半后台框架的团队，终于想明白了一件事：开发者不需要"全家桶"，需要的是"乐高"。

这次 Forge Admin 的更新，其实就围绕一句话：**从"大而全"到"可组合"**。过去我们习惯把功能塞满，生怕用户不够用；后来发现，真正用起来的开发者，反而花大量时间在"关掉不需要的东西"上。所以我们这次做了一件反直觉的事——把框架拆开，做成底座+插件化架构。引擎归引擎，业务归业务，你只需要装自己要用的。

下面是这次更新的四个重头戏。

---

## 一、流程引擎：审批不只是"点一下通过"

以前 Forge Admin 的流程引擎能跑，但也仅仅是"能跑"。这次我们从设计器到用户体验全部重构了一遍。

**可视化流程设计器，拖拽就完事。** 不再需要对着 XML 或者 JSON 配流程节点，直接在画布上拖、拉、连线。条件分支、并行审批、会签、转审，所见即所得。以前配一个复杂报销流程要 20 分钟，现在 5 分钟画完。

<!-- screenshot: 流程设计器拖拽界面 -->

**审批时直接填业务表单。** 这是这次最"爽"的改动。过去审批人在审批界面和业务表单之间来回跳——先看一眼报销金额，再切到审批页点通过，再切回来核对明细。现在流程节点直接绑定动态表单，审批人打开待办就看到完整的业务数据，审批、填写、驳回在同一页面完成。不用跳、不用等、不用切窗口。

<!-- screenshot: 动态表单配置界面 -->

**内置流程测试，不用部署就能跑。** 配完流程，点一下"测试运行"，模拟不同角色走一遍全流程。哪个节点卡住了、哪个条件分支走错了，当场看到、当场改。不用像以前那样部署完才发现审批顺序写反了。

<!-- screenshot: 流程测试运行界面 -->

**消息通知全覆盖。** 站内通知 + 企微/飞书/钉钉推送，审批人在哪儿都收得到。审批超时还能自动催办，不用 PM 拿着 Excel 到处追人。

**审批界面重新设计。** 待办、已办、我发起的，三栏布局一目了然。右侧是流程进度条，当前节点高亮，已经过的节点灰色，一眼看明白卡在哪儿。

<!-- screenshot: 审批待办列表界面 -->

## 二、低代码应用管理：乐高模式，你需要什么就装什么

如果说流程引擎是"强化"，那低代码应用管理就是"重构"。这次的核心变化：**引擎和业务彻底分离**。

我们把 Forge Admin 拆成了四层：

- **核心引擎层（底座）**：流程引擎、审批引擎、报表引擎、权限引擎、消息引擎。这是地基，不管你做什么业务，这层都要有。
- **应用层单据**：CRM、订单管理、合同管理、项目管理……每个都是一个独立的应用包，按需安装。
- **移动端 H5**：独立入口，业务流程移动端全覆盖。
- **外部扩展**：标准开放接口，推送到企微、飞书、钉钉。

<!-- screenshot: 应用管理-应用市场列表 -->

**为什么要这么拆？** 讲一个真实场景：你是一家贸易公司，初期只需要 CRM 管理客户和商机。在以前的版本里，你打开 Forge Admin，左边菜单栏密密麻麻：订单、合同、项目、进销存、工单……你实际只用 CRM 一个模块，但其他 20 个模块的菜单就在那儿碍眼。

现在？打开应用市场，装一个 CRM，菜单栏就只出现 CRM 相关内容。界面干净。

<!-- screenshot: CRM应用界面 -->

三个月后业务跑起来了，老板说要数据大屏。你在应用市场找到"大屏看板"，一键安装。报表引擎自动拉取 CRM 数据，嵌入到看板里，当天下午老板就在会议室投屏看数据了。

<!-- screenshot: 大屏看板嵌入式展示 -->

我们用一句话概括这个设计哲学：**引擎是引擎，业务是业务，不再搅在一起。** 企业拿到的是一盒乐高，而不是一整块水泥。

移动端 H5 也同步上线，从应用到审批到消息，手机端完整闭环。

<!-- screenshot: 移动端H5界面 -->

## 三、多租户管理：这次把"上层建筑"补齐了

Forge Admin 的五级租户忽略策略一直是核心竞争力——系统表自动跳过租户隔离，业务表自动注入 tenant_id，开发者不用在每个 SQL 里手写 `WHERE tenant_id = ?`。但之前租户管理的上层能力比较薄，这次补了三个关键对象：

**组织管理：支持多级组织树。** 集团 → 子公司 → 部门 → 岗位，任意层级嵌套。人员归属清晰了，审批流里的"部门负责人"才知道到底是谁。

<!-- screenshot: 组织架构树管理界面 -->

**用户管理：批量导入 + 租户间隔离 + 登录日志追踪。** 100 个员工入职，Excel 一拖全导入，不用一个一个加。谁在什么时间从哪个 IP 登录、做了什么操作，全部可追溯。

<!-- screenshot: 多租户用户管理界面 -->

**角色管理：角色与数据权限联动。** 同一个"销售经理"角色，在华东租户能看到华东的客户数据，在华南租户就只能看到华南的。权限粒度从"功能级"下沉到了"数据行级"，而且是可视化配置，不是改 XML。

<!-- screenshot: 角色-数据权限联动配置 -->

## 四、移动端来了

过去很多后台框架的"移动端"就是网页缩放一下，表单挤成一团，按钮小到手指点不中。

Forge Admin 这次上了独立移动端 H5 入口，自适应布局，业务表单在手机端和 PC 端体验一致。审批流程、待办提醒、消息通知，手机上全搞定。

支持与企业微信/飞书/钉钉的移动端入口打通，员工不用额外装 App，从 IM 里直接点进审批页。

<!-- screenshot: 移动端待办列表 -->

<!-- screenshot: 移动端审批操作界面 -->

## 最后

这次更新，我们做的最多的不是"加功能"，而是"拆结构"——把引擎和业务拆开，把底座和扩展拆开。因为相信一个道理：**好的框架，是让开发者少做事，而不是替开发者决定要做什么。**

你拿到的是乐高，不是水泥块。
