# 测试方案 — 低代码设计器触发器、表格居中与 ER 图优化

## 1. 测试范围

- 对象设计器内嵌触发器面板。
- 独立触发器配置页回归。
- 低代码列表列默认居中。
- 关系与级联 ER 图展示。

## 3. 手工烟测

| 编号 | 场景 | 步骤 | 预期 |
| --- | --- | --- | --- |
| M1 | 对象设计器触发器面板 | 打开任一低代码对象设计器，点击左侧“自动化触发器” | 页面不跳转，触发器列表按当前对象过滤 |
| M2 | 新增对象触发器 | 在内嵌面板点击“新增触发器” | 业务单元为当前对象且不可切换，保存调用现有接口 |
| M3 | 独立触发器页 | 打开 `/app-center/trigger` | 可筛选业务单元、场景，新增时可选择业务单元 |
| M4 | 表格默认居中 | 打开低代码列表设计器预览未配置对齐的列 | 表头和内容居中；显式设置对齐的列不被覆盖 |
| M5 | ER 图 | 在关系与级联面板点击 ER 图入口 | 展示当前对象节点、关系连线和下载 SVG 按钮 |

## 4. 回归关注

- 内嵌触发器不应污染路由查询参数。
- 触发器执行日志抽屉仍可打开并分页。
- 对象设计器保存/发布按钮不受触发器面板影响。
- ER 图目标对象字段加载失败时，不阻塞关系配置原有功能。
