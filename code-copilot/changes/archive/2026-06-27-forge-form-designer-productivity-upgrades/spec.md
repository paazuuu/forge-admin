# Forge 原生表单设计器效率增强
> status: done
> created: 2026-06-17
> scope: `forge-admin-ui`
> related: `code-copilot/changes/lowcode-form-designer-smooth-dnd/spec.md`

## 1. 背景

Forge 原生表单设计器已经替代 FormCreate 作为业务对象表单主画布，但仍存在几个影响日常配置效率的问题：

- 拖拽到非法位置时静默失败，用户不知道是没拖准还是该容器不支持。
- 表单设计没有撤销/重做，误删、误拖后只能手工恢复。
- 右侧属性面板内容越来越多，用户找“字典”“默认值”“校验”等配置成本高。
- 预览只有单一状态，无法快速检查新增、编辑回显、详情只读三种运行态效果。

## 2. 本轮目标

- 拖入非法位置时给出明确提示，并区分根区域、容器内部、同节点/后代节点等情况。
- 在 ForgeFormDesigner 内增加会话级撤销/重做历史栈，覆盖画布拖拽、属性修改、复制删除、表单设置等 schema 变更。
- 右侧属性面板增加属性搜索，输入关键词后自动切换到相关页签、展开相关折叠卡片，并高亮匹配项。
- 预览弹窗增加三种模式：新增、编辑、详情。编辑模式用 schema 默认值和模拟详情数据回显，详情模式只读展示。

## 3. 非目标

- 不改后端接口协议。
- 不新增数据库脚本。
- 不重构完整组件注册表，本轮只在现有结构内做低风险增强。
- 不实现多人协同、持久化历史版本或跨会话撤销。

## 4. 后续优化方向

后续应继续推进以下架构级收敛：

- 建立统一组件注册表，集中管理 `componentKey`、别名、分类、默认 props、属性配置、设计态渲染和运行态渲染。
- 让画布预览、弹窗预览和发布运行态共用同一套 renderer，减少“画布正常、预览异常、发布后又不一致”的问题。
- 拆分 `ForgePropertyPanel.vue`，把字段、布局、CRUD、交互、源码和表单资产配置拆成独立组件。
- 建立设计器自检面板，保存/预览/发布前检查字典、字段绑定、按钮事件、弹窗表单、CRUD API 等配置风险。
- 继续完善拖拽体验：原位幽灵、跟手镜像、落点双胞胎阴影、非法落点提示和减少动态效果兼容。
- 扩展模板片段、AI 辅助生成、移动端 Renderer、权限/流程联动配置。

## 5. 验收标准

- 将字段拖入不支持接收的位置时，画布出现明确的非法提示。
- 点击撤销后能恢复上一步 schema，点击重做能恢复撤销前 schema。
- 修改属性后再撤销，右侧和画布都回到修改前状态。
- 在属性面板搜索“字典”“默认值”“校验”“CRUD”“弹窗”等关键词能定位对应配置区域。
- 预览弹窗可切换新增、编辑、详情；详情模式字段只读，编辑模式能看到模拟回显值。
- 前端构建通过；如 lint 因既有格式问题不能通过，需要说明原因。
