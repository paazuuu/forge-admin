# 测试方案 — 低代码设计器工作台视觉统一

## 1. 测试目标

验证表单设计器和列表设计器在只改布局、样式和展示交互后，核心业务功能不回退，且两者视觉风格统一。

## 2. 自动化验证

- `pnpm --dir forge-admin-ui exec eslint ...`
  - 覆盖本次涉及的 Vue 文件。
- `pnpm --dir forge-admin-ui build`
  - 验证 Vite 构建通过。
- `git diff --check`
  - 验证无空白字符问题。

## 3. 手工验收场景

- 表单设计页
  - 页面打开后左侧组件库、中间画布、右侧属性面板高度一致。
  - 页面切换条为统一样式，能切换主表单和表单资产。
  - 鼠标滚轮缩放画布可用。
  - 点击视口/宽度/缩放悬浮控制能打开原有配置。
  - 拖入字段、拖入布局、选中组件、修改属性、撤销重做、清空画布入口可用。
  - 拖拽时右侧属性面板自动收起行为保留。

- 列表设计页
  - 顶部工具栏与表单设计页风格一致。
  - 页面切换条不再挤占过多空间，配置项可通过折叠或弹出方式访问。
  - 中间画布点阵背景、白色页面和悬浮控制与表单一致。
  - 拖入区块、移动区块、调整区块大小、选中区块、修改属性可用。
  - 页面配置、参数配置、数据配置修改后仍能同步到原数据结构。

- 响应式
  - 1366x768：核心画布可见，按钮不乱换行。
  - 1440x900：左右面板和中间画布比例稳定。
  - 1920x1080：白色页面居中，页面切换条信息不拉散。

## 4. 不测试范围

- 后端接口。
- 数据库迁移。
- 运行态 CRUD 业务功能深度回归。
- 拖拽接收规则和网格算法本身的新增能力。
