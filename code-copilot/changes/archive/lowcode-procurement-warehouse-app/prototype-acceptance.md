## 2. 验收结论

当前版本已经具备通过完全低代码方式搭建采购仓储模块的主业务能力：

- 采购、仓储、供应商、物料四类入口已落到应用中心运行页。
- 采购单、出库单、调拨单具备主子表明细维护。
- 供应商报价、采购明细、出库明细、调拨明细具备记录选择器批量带入。
- 采购/出库/调拨已配置流程提交按钮和审批回调动作。
- 数量余额、数量流水、数量锁定通过通用数量台账能力展示和流转。
- 原型演示数据已 seed，便于进入页面后做人工验收。

仍未做到像素级 1:1 的部分主要是平台 UI 模板层能力，不是业务逻辑缺口。

## 4. 暂需后续平台 UI 能力

- 原型里的卡片式仓库统计，例如物料种类、库存总量、预警物料、本月入库/出库，目前用数量表格等价表达；后续可补“详情统计卡片面板”。
- 原型里的底部固定操作栏、局部按钮排版、紧凑表格视觉，目前使用通用 `AiCrudPage` 操作列和表单弹窗。
- 出库/调拨选择物料时按仓库真实可用库存过滤候选，目前已具备仓库上下文透传和数量余额展示，候选源仍以物料/报价对象为主；后续可增加“库存余额选择器对象”。
- 状态标签、仓库类型、单据状态目前以配置值显示；后续可补采购仓储内置字典，让标签颜色和文案更贴近原型。

## 5. 人工验收建议

1. 运行 Flyway 后进入应用中心，确认存在“采购仓储”套件。
2. 打开物料、供应商、仓库、采购、出库、调拨运行页，确认 demo 数据可见。
3. 打开仓库详情，确认库存余额、数量流水、数量锁定和关联单据面板可加载。
4. 打开物料详情，确认供应商报价、库存信息和近 3 次数量流水可加载。
5. 在出库单点击“提交审批并锁定”，在采购/调拨单点击“提交审批”，确认动作能进入通用审批链路。
