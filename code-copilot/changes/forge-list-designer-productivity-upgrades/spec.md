# Forge 列表设计器效率与体验增强
> status: apply
> created: 2026-06-18
> scope: `forge-admin-ui`
> related: `code-copilot/changes/forge-form-designer-productivity-upgrades/spec.md`

## 1. 背景

表单设计器已经完成一轮原生画布体验增强，形成了较稳定的交互范式：紧凑工具栏、更多菜单、撤销/重做、右侧配置自动展开、保存前置 loading 和更明确的拖拽反馈。

列表设计器当前仍偏“配置面板 + 预览区”：

- `BusinessListDesigner.vue` 只负责模式切换、树形布局开关、保存和 Schema 同步，缺少历史栈和统一工具栏。
- `StructuredListPageDesigner.vue` 已有查询字段和列表字段配置，但字段排序藏在弹窗里，整体与表单设计器视觉节奏不统一。
- `ListPageGridDesigner.vue` 已支持区块拖拽、移动、缩放和属性抽屉，但缺少外层撤销/重做、低频操作收纳和更明确的状态反馈。

## 2. 可复用分析

可以复用表单设计器的交互模式：

- 会话级撤销/重做历史栈：列表设计器同样是 JSON Schema 编辑，适合快照式历史。
- 紧凑工具栏和更多菜单：把低频操作收进更多，保留高频状态开关。
- 选中即配置：自由布局已具备选中区块打开属性抽屉，可继续强化。
- 保存 loading 前置：外层对象设计器已处理，列表内保存也应保持即时反馈。
- 拖拽视觉语言：hover、选中、拖动手柄、落点提示可以逐步向表单设计器靠拢。

不直接复用的部分：

- `ForgeFormCanvas` / `ForgeFormCanvasNode` 不直接服务列表，因为列表编辑对象是 `PageSchema/ViewSchema` 的 zone/block，不是 `FormDesignerSchema.components`。
- `ForgePropertyPanel` 不直接复用，因为列表区块属性和字段属性与表单组件属性不同。

## 3. 首轮目标

- 为 `BusinessListDesigner.vue` 增加会话级撤销/重做，覆盖结构化配置和自由布局 schema 更新。
- 标题区压缩为单行，视觉节奏对齐表单设计器。
- 将低频列表操作放入更多菜单，减少头部挤压。
- 将结构化列表配置从纯弹窗字段配置升级为“左预览 + 右字段配置”的工作区。
- 为自由布局画布增加拖拽落点反馈，并避免移动/缩放过程产生过多历史快照。
- 将自由布局的区块属性配置从抽屉改为固定右侧配置面板，避免遮挡画布和反复弹出。
- 将自由布局区块缩放从单一右下角拖拽改为八方向锚点缩放，支持上下左右和四角调整。
- 为自由布局增加左右面板收起/展开，选中画布区块时自动展开右侧配置区块。
- 将区块移动入口收敛到顶部拖动手柄，并将复制、层级、移动、删除收纳到区块右上角更多操作。
- 清理区块选中态的重复边框，外层使用实心缩放锚点，内部预览仅保留轻量选中提示。
- 统一自由布局画布、区块库、配置区块的视觉层次，减少拥挤和单一底色。
- 左侧区块库补充真实系统组件区块：`AiCrudPage`、`AiTable`、`AiForm`，并在画布中按真实组件预览。
- 自由布局拖动命中其它区块时支持位置交换，避免只能下挤、不能上下换位。
- 强化画布边界、选中边框、更多操作入口和右侧属性 label 对齐。
- 自由画布作为列表最终效果编辑主模式；结构化配置降级为“快速配置”，只承担批量字段和参数整理。
- 自由画布列宽随中间区域自适应，避免右侧出现不可拖拽、不可缩放的空白区域。
- 区块移动过程只更新视觉位移和落点色块，松手后再提交 schema，减少拖拽卡顿和历史栈噪声。
- 自由画布提供“清空当前画布”，只清空当前列表画布区块，不删除字段模型和其它设计面板。
- 空画布必须作为有效布局保存和同步，`items: []` 不能被 `syncGridLayoutWithModel` 重新回填默认区块；逐个删除到最后一个区块时同样保持空画布。
- 区块库支持按区块名称、说明和组件 key 搜索；画布工具栏提供源码查看，便于对照当前布局 JSON 和选中区块 JSON。
- 配置字段弹层需要保证下拉、字段设置和代码列不撑出抽屉，字段配置应以可换行的卡片流展示。
- 左侧入口命名为“页面组件”，不能只表达固定业务区块；组件库需要包含通用页面布局组件。
- 所有页面组件默认 `width: 100%`、`height: 100%`，并支持背景色、边框色、边框宽度、边框样式、圆角、阴影、内边距和自定义 CSS style。
- 页面组件库补充常用布局组件：卡片、Tabs、横向/竖向分隔线、留白占位。后续新增组件应优先作为通用页面组件扩展，而不是写死业务页面结构。
- 卡片和 Tabs 不是纯展示壳，必须支持承载子组件；首期采用轻量 children 列表承载，拖入容器后按子组件列表渲染，后续再升级为容器内 12 栅格。
- 每个页面组件都必须保留通用事件配置入口，事件统一落到 `props.events`，字段包含触发时机、动作、目标组件、说明和参数映射。
- 左侧导航树的“节点选择联动右侧表格/组件”也走通用事件模型，推荐事件为 `nodeSelect -> filterBlock`，并显式配置目标组件和过滤字段。
- 列表设计器支持页面级多画布，默认包含“列表页”和“详情页”；详情页默认包含返回上一页组件和详情内容区块，后续按钮、行操作或事件可引用这些页面。
- 页面级多画布必须支持页面管理：新增、复制、删除自定义页、重命名、修改页面编码、页面类型、页面路径、页面说明和页面入参。
- 默认详情页允许删除；删除后通过 `removedPageKeys` 记录，不应在保存、刷新、同步 schema 时自动补回。
- 默认详情页不使用 `AiForm` 作为展示内容。`AiForm` 定位为新增/编辑弹窗内容，详情页默认使用只读 `detail-info` 详情信息组件。
- 页面编码修改时需要同步修正页面组件事件中的 `targetPageKey` 引用，避免已有跳转事件失效。
- 当前页面支持“重置布局”和“清空布局”：重置回到该页面类型的默认模板，清空保留页面资源但清空 `gridLayout.items`。
- 通用事件配置需要同时支持目标组件和目标页面。目标组件用于刷新/过滤/联动，目标页面用于页面跳转、详情页、弹窗页和抽屉页。
- 左侧页面组件使用两列布局展示，组件命名以页面组件为主，避免把通用布局能力误表达为固定业务区块。
- 页面组件默认不附带边框和圆角，避免拖入组件后每个组件都有多余包裹感；需要卡片化、边框、圆角、阴影时通过右侧外观配置显式设置。
- 页面组件库应尽量覆盖常用页面搭建元素：页面标题、返回上一页、详情信息、按钮、按钮组、提示面板、标签列表、步骤条、时间线、空状态、卡片、Tabs、分隔线、留白、系统 AiCrudPage/AiTable/AiForm。
- 系统组件 `AiCrudPage`、`AiTable`、`AiForm` 需要暴露常用 props，包括标题、字段、显示项、布局、接口、搜索、表格、弹窗、工具栏和事件回调；接口配置必须使用结构化表单项，不能要求用户手写 JSON 或猜回调方法名。
- 系统组件在画布中不展示额外说明头，例如 `AiForm / 系统表单组合`，应直接预览真实组件内容，避免重复包裹影响用户判断。
- 页面组件外观配置需要区分“设计画布显示框”和“运行时 CSS 尺寸”。中间设计画布默认宽度为 1366px，并允许在工具栏动态调整为 1200、1440、1600、1920 或手输宽度；组件默认 `widthMode: full` / `width: 100%`，并支持切换为 `auto` 或固定 px；每个组件仍能单独配置 height、min/max、padding、margin 和自定义 style。
- 自由画布的主要交互单位应是 px，而不是 12 栅格列/行。`gridX/gridY/gridW/gridH` 只作为兼容和发布投影字段保留，设计器拖拽、缩放和右侧位置尺寸编辑应基于 `props.style.x/y/widthMode/width/height`。
- 新建或重置列表页默认使用一个配置好的 `AiCrudPage` 作为主内容区，字段来自当前业务对象字段，用户进入后应是“默认能用，局部调整”的状态，而不是空白画布或拆散的查询/工具栏/表格组合。
- 多页面的 `routePath` 当前不是独立前端路由文件路径，而是统一业务对象运行页内部使用的路由片段/页面标识，用于事件跳转、详情页、弹窗页、抽屉页识别；UI 文案不能误导用户以为每个页面都会生成独立路由。
- 画布模块 hover 时只显示明显区域反馈，例如外框高亮和阴影；不显示组件名称浮标，避免遮挡真实页面内容。
- 多页面切换区需要降低理解成本，提供“页面设计”标题、说明、页面类型颜色提示，并隐藏横向滚动条避免 hover 时滚动条闪动。
- 顶部预览按钮必须走本地即时预览：列表页直接打开当前画布全屏预览，不调用保存草稿、发布检查或 `previewBusinessObjectLayout`。
- 列表页预览不能被表单设计器的预览事件拦截，表单和列表分别使用自己的本地预览事件。
- 表格列不能只停留在固定文本列。字段配置需要支持无代码列渲染：渲染方式、文字颜色、点击动作、目标页面等，典型场景是“编码列显示为蓝色链接，点击跳转详情页”。
- 自定义插槽类需求不直接暴露代码插槽给普通用户，优先抽象为“渲染模板 + 样式 + 点击动作 + 参数映射”。高级用户后续可再扩展为受控模板/脚本，但默认链路必须无代码可配置。
- 当前业务运行页仍是统一路由，如 `/ai/crud-page/:objectCode`。列表设计器里的多页面不是生成多个前端路由文件，而是保存到 `pageSchema.pages[]`，运行态需要在统一路由内部解释 `targetPageKey`、页面参数和页面类型，完成列表页、详情页、弹窗页、抽屉页之间的内部导航。
- 列表列跳详情需要显式支持传参配置，首期提供单参数配置：目标页面、参数名、取值字段。默认使用 `id = row.id`。
- 详情页需要有页面级详情数据源配置，至少包含请求方式、详情接口和响应数据字段。详情组件运行时应优先消费当前详情页数据源，不要求用户在只读详情组件里手写请求逻辑。
- “CRUD配置”不能成为第二套独立设计器。它只作为默认 `AiCrudPage` 的快速字段配置入口，改查询字段、列表字段、排序、列宽和列属性时必须同步到自由画布中的 `AiCrudPage` / `listGridLayout` / `pages.list.gridLayout`。
- 保存发布前必须以当前自由画布 `listGridLayout` 和 `pages.list.gridLayout` 为准回写 `zones` 和 `viewSchema`，避免用户设计完成后应用页面仍使用旧结构化配置。
- 字段配置区域不能只靠 placeholder 解释含义。跳转参数、映射字段、列宽、固定列等配置必须有短标签和必要说明，避免用户看到默认 `id` 却不知道它代表什么。
- 保持结构化/自由布局现有数据协议不变，避免影响发布运行态。
- 页面组件库继续扩展组件时必须沿用当前工作台视觉：左侧两列组件卡片、中间真实预览、右侧“属性 / 样式 / 交互”配置；所有新增组件必须保留 `props.style` 通用外观和 `props.events` 通用事件入口。
- 本轮新增页面组件需求包括：手写签名、富文本框、穿梭框、分步表单、Vue 组件占位、HTML 标签、标题、段落、统计数值、链接、文字提示、水印、音频播放器、视频播放器、头像框、条形码、内嵌页面、二维码、Markdown、盒子布局、间距和描述列表。
- 新增组件默认应提供丰富可配置属性，至少覆盖内容、尺寸、状态、数据源/绑定字段、展示样式、交互行为和安全提示；能用结构化配置表达的能力不能要求普通用户手写 JSON。
- 富文本、Markdown、HTML、内嵌页面、Vue 组件和媒体类组件必须有安全边界：默认以预览/受控内容渲染为主，运行态涉及原始 HTML、外链 iframe、动态组件名、媒体地址时要保留白名单、只读预览或后续安全审查入口，禁止默认执行任意脚本。
- Vue 组件、Markdown、富文本和 HTML 标签不能只是占位卡片：Vue 组件需要可配置组件代码/模板/脚本/样式和 props 示例，并以安全预览展示；Markdown 需要源码编辑和解析预览；富文本需要工具栏式编辑能力；HTML 标签需要标签名、属性、文本/HTML 内容、语义和安全策略配置。
- 页面组件能力必须同时服务列表设计器和表单设计器。新增通用展示/媒体/嵌入类组件应抽取共享 schema、默认 props、渲染器和属性编辑模型，列表设计器与表单设计器只做适配，禁止长期复制粘贴两套实现。
- 二维码、条形码、手写签名如果不新增依赖，首期可先实现设计态占位和结构化 schema；真实生成/采集能力优先复用项目已有组件或后续引入经过评估的轻量依赖。
- 栅格布局应支持拖入子组件后自动根据子组件内容和数量撑高，避免用户把组件拖入栅格后内容被裁切；自动撑高应只在内容增长或默认高度不足时生效，用户手动设置固定高度后不能被反复覆盖。

## 4. 后续方向

- 多画布已进入列表设计器主链路，首期采用 `pageSchema.pages[]` 存储列表页、详情页和自定义页面；后续需要补充页面树、页面参数、动作目标选择器和运行态路由/弹窗承接。
- 页面管理首期保护内置 `list/detail` 页面不允许删除；自定义页面可删除。后续如果产品需要删除默认详情页，需要调整 `ensureDesignerPages` 的补齐策略。
- 容器 children 首期为轻量列表，后续可升级为容器内独立 12 栅格、容器内选中子组件直接编辑属性、容器内拖拽排序和跨容器移动。
- 事件配置首期只负责 schema 表达和设计期配置，后续需要由运行态统一解释 `props.events`，支持页面跳转、刷新组件、过滤组件、打开弹窗/抽屉、接口请求和自定义脚本。
- 继续细化 `StructuredListPageDesigner` 的字段配置面板，逐步拆出独立组件并补充字段搜索、批量选择和非法配置提示。
- 把 `ListPageGridDesigner` 的 palette、canvas、property drawer 风格统一到 Forge 表单设计器。
- 增加非法 drop 提示和落点预览，尤其是区块唯一性、树形区块布局限制等场景。
- 增加列表预览模式：查询态、空数据态、有数据态、树形态。
- 抽取通用 `useDesignerHistory`，供表单、列表、详情设计器复用。

## 5. 验收标准

- 列表设计器修改字段、切换结构化/自由布局、调整树形布局后可撤销/重做。
- Ctrl/Cmd+Z 和 Ctrl/Cmd+Y / Shift+Ctrl/Cmd+Z 可用于列表设计器历史操作。
- 头部高度更紧凑，说明文字不撑开多行。
- 前端定向 lint 和构建通过。
- 自由布局中区块拖动、缩放、更多操作、左右面板收起展开可正常使用。
- `AiCrudPage`、`AiTable`、`AiForm` 作为列表整体型区块可从区块库拖入，并能配置字段。
- 自由画布默认进入，画布宽度铺满中间工作区；宽度为 12 列的区块需先缩窄后才能左右移动。
- 拖动时有落点色块，拖动过程不卡顿，松手后再完成换位或落位。
- 点击顶部预览按钮直接打开当前列表本地预览弹窗，不保存、不校验、不切发布检查。
- 卡片/Tabs 中拖入或添加子组件后，画布和预览都能看到子组件内容。
- 切换到详情页时能看到独立详情页画布，默认存在返回上一页组件。
- 默认详情页能删除，删除后保存或刷新不会被自动补回。
- 详情页默认展示组件是 `detail-info` 只读详情信息，不是 `AiForm` 表单输入态。
- 拖入页面组件后默认没有边框和圆角；右侧配置里仍可手动开启边框、圆角、背景和阴影。
- 左侧组件库包含常用页面组件，并可在右侧配置主要属性。
- 多页面管理区可新增、复制、删除自定义页面，并能配置页面名称、编码、类型、路径、说明和入参。
- 重命名自定义页面编码后，已配置的事件目标页面引用同步更新。
- 任意选中区块都能在右侧维护通用事件；树区块能生成节点选择联动目标组件的事件。
- 系统组件右侧配置不能出现“接口配置 JSON”“回调方法名”这类需要猜格式的输入，`AiCrudPage` 至少应按折叠组暴露基础/API、搜索与表格、表单与弹窗、工具栏与导入导出、事件说明。
- `AiCrudPage`、`AiTable`、`AiForm` 画布预览应消费右侧配置项，配置后能立即影响预览中的显示项、尺寸、模式和布局。
- 右侧“位置与尺寸”显示 px 坐标和 px 宽高，拖拽/缩放过程应连续跟手，不应按列/行跳动。
- 新建列表页或重置列表页后，中间画布默认出现 `AiCrudPage`，并带当前模型字段。
- 页面路径配置说明明确为路由片段/页面标识，不能再表达成真实页面路径。
- 鼠标经过画布模块时，有 hover 边框和阴影反馈，但不出现组件名称浮标。
- 中间设计画布默认宽度为 1366px，并可动态调整；新建、重置和旧默认 12 栅格组件不应再保存成固定 `1024px` 宽，默认运行时宽度为 `100%`。
- 列表预览弹窗应全屏显示并支持滚动；预览态 `100%` 宽度按预览容器计算，不按设计画布宽度计算，只有固定 px 宽度组件保持固定。
- 列表字段配置中可把某一列设置为链接文本、设置文字颜色、选择目标页面、配置参数名和取值字段；设计器预览中能看到链接样式，schema 中保存 `renderType/clickAction/targetPageKey/textColor/targetParamName/targetParamField` 等配置。
- 详情页配置区能维护详情接口、请求方式和详情数据字段。
- 切换到 CRUD 配置后，页面文案应明确这是默认 CRUD 组件的快速配置，不是另一套页面设计；字段配置弹层和右侧面板不能出现下拉超出容器。
- 保存列表后，发布应用页面应拿到由自由画布同步出的 `zones/viewSchema`，不能因为结构化/自由布局模式切换导致页面不更新。
