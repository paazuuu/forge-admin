# AI Generate P2 Optimization Spec

## 背景

P0/P1 已经完成前端生成校验、业务准备度、数据集画像和自动绑定修复。当前仍存在一个关键缺口：AI 大屏生成记录只保存在浏览器本地，刷新、换设备或排查生成质量时缺少后端可追溯数据。

## 目标

1. 后端持久化 AI 大屏生成记录，保存请求、业务定义、模型、生成结果和校验统计。
2. report-ui 在生成成功、解析失败、生成失败或用户停止时写入生成记录。
3. AI 助手历史面板优先展示云端记录，本地历史作为兜底。
4. 生成记录可直接重新应用到当前画布，失败记录展示错误原因但不可应用。

## 范围

### In Scope

- `forge-plugin-ai` 新增 AI 大屏生成记录表、实体、Mapper XML、Service 和 Controller。
- `forge-report-ui` 新增生成记录 API、保存生成结果、读取云端历史并复用结果。
- P2 变更文档和构建验证。

### Out of Scope

- 后端自动解析模型流式响应内容。
- 管理端全量生成质量分析页面。
- Prompt 模板版本管理和二次 AI 修复循环。

## 验收标准

1. AI 生成成功后，后端能保存生成标题、组件数、动态绑定数、静态降级数、修复数和完整响应 JSON。
2. AI 生成失败、停止或 JSON 解析失败时，后端能保存失败状态和错误原因。
3. 打开 AI 助手最近生成记录，能看到云端历史；成功记录可以再次应用到画布。
4. 相关后端 SQL 查询写在 Mapper XML 中。
