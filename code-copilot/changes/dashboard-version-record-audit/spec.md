# 大屏版本管理与生成记录稽核

## 背景

当前大屏编辑器启用了实时保存，编辑过程会持续写入 `ai_report_project.component_data`。如果把版本创建绑定到普通保存，会产生大量无业务意义的版本记录，影响历史版本查看和回退体验。

管理侧已有 `ai_dashboard_generate_record` 表保存 AI 大屏生成记录，但缺少稽核查询页面和跨用户分页接口。

## 范围

- `forge-report-server` 新增大屏发布版本表、实体、Mapper、服务和接口。
- `forge-report-ui` 新增历史版本查看、回退、下载配置、历史版本预览入口。
- `forge-plugin-ai` 新增生成记录管理端分页/详情接口。
- `forge-admin-ui` 新增生成记录稽核页面。
- 数据库增量脚本放入 `forge/db/migration`。

## 规则

1. 普通实时保存只更新当前草稿，不创建历史版本。
2. 发布成功后创建版本快照，记录项目配置、发布地址、发布人、发布时间。
3. 回退历史版本时恢复项目配置，并追加一条回退版本记录，保留审计链路。
4. 历史版本预览读取版本快照，不污染当前项目草稿。
5. 生成记录稽核接口按租户查询，可按项目、业务定义、用户、状态、时间过滤。

## 验收标准

- 发布大屏后可在版本弹窗看到版本号、发布人、发布时间、操作类型。
- 历史版本支持预览、下载 JSON 配置、回退。
- 编辑器实时保存不会新增版本记录。
- 管理端可查询 AI 大屏生成记录并查看 prompt、生成摘要、错误信息等详情。
- 后端编译通过，前端构建或 lint 至少完成相关项目验证。
