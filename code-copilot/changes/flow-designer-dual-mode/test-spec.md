# 测试计划 — 流程模型双设计器模式改造

## P0 增量验证
- `git diff --check`：检查空白错误，最终通过。
- 目标 ESLint：覆盖恢复的 BPMN 组件、流程模型列表、流程设计页，最终通过。
- 前端构建：确认 `bpmn-js` 依赖、恢复组件和双模式设计页可编译，最终通过。
- 后端 flow 插件编译：确认 `FlowModel` 新字段不影响编译，最终通过。
- 业务流程画布回归：空 `bpmnXml`、路由切换到新流程、版本刷新空 XML 均应导入默认空 BPMN 模板，不沿用上一个模型画布；BPMN 画布自适应缩放不得超过 100%。

## P1 交互验证
- 打开流程模型新建弹窗，确认可选择“审批流程/业务流程”。
- 创建或编辑审批流程时加载钉钉式设计器。
- 创建或编辑业务流程时加载 BPMN.js 专业设计器。
- 在业务流程画布点击“适应屏幕”或执行自动布局时，单个开始节点不被放大，自动布局保持左到右方向。
- 点击 BPMN 业务流程节点时右侧属性面板稳定停靠，不因节点切换产生反复滑入或闪烁；审批流程和业务流程属性面板使用统一外壳。
- 属性面板样式布局一致性：审批流程与业务流程面板使用一致的宽度、顶部标题区、滚动容器、Tab 间距、表单 label、输入框高度和内容 padding；不改参数映射、字段写入和转换逻辑。

## 跳过项
- 不做真实数据库迁移实跑：本轮提供 Flyway 脚本，后端编译覆盖实体变更；实际迁移由环境启动时执行。
- 不做真实流程部署联调：两套设计器最终仍保存 `bpmnXml`，部署链路沿用现有逻辑。
- 不做浏览器登录态交互验证：本轮未启动本地后端和数据库，前端通过目标 ESLint 与生产构建覆盖编译和依赖解析风险。
