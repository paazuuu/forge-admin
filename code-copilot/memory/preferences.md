# Forge项目用户偏好

## 1. 前端 Node 版本

**记录日期**: 2026-05-07

运行前端相关命令（如 `pnpm install`、`pnpm exec eslint`、`pnpm build`）前，先执行：

```bash
source ~/.nvm/nvm.sh && nvm use v20.19.0
```

避免使用系统默认 Node 12 导致 pnpm 版本不兼容。

## 2. 前端样式变更偏好

**记录日期**: 2026-05-14

用户对“看起来像 AI 生成”的后台页面较敏感，尤其反感只改 CSS 但视觉无明显变化的结果。涉及页面美化时需要做可感知的结构、布局和交互改动，并通过 lint/build 或本地预览验证后再反馈。

## 3. 自动化测试执行偏好

**记录日期**: 2026-06-02

用户希望自动化测试流程标准化、可复用。后续执行 `/test`、阶段验证或归档前验证时，必须先读取 `code-copilot/rules/automated-testing-standard.md`，复用当前变更已有 `test-spec.md`、`execution-log.md`、`spec.md`、`tasks.md`，根据本轮差异做增量验证，不要每次从零开始。

## 4. 低代码表单设计器交互偏好

**记录日期**: 2026-06-15

用户希望低代码表单设计器前端操作便捷简单，拖拉拽体验要接近飞书多维表格的顺滑感；拖动过程需要有精心设计的动画反馈，包括跟手镜像、原位幽灵、落点提示和“后面的双胞胎阴影”，不能只是基础拖拽和静态样式调整。

用户特别强调布局必须合理、组件必须细分、注释和逻辑必须清晰、交互必须简单一目了然。可拖拽区域在鼠标 hover 时必须有明显反馈，例如拖拽图标、手柄、背景/边框变化或鼠标样式变化，不能让用户猜测哪里可以操作。

字段相关配置优先采用飞书多维表格风格：属性面板外层展示摘要行（如“展示字段 + N 个字段 + 设置图标”），点击设置后在浮层中展示字段行；字段行应包含拖拽手柄、字段类型图标、字段名/编码和显示/隐藏等图标按钮，避免把大量开关或字段列表直接堆在属性面板中。

## 5. 首页社区入口与登录页动效偏好

**记录日期**: 2026-06-28

登录页需要优先保证低端设备流畅，不要添加无业务价值的持续动画、扫光、流光边框、重滤镜或大面积模糊效果。首页可以放社区运营入口，但二维码区域必须明显、可点击放大；涉及收款/打赏/维护支持时，文案要委婉，优先使用“支持维护”“请维护者喝杯咖啡”“随心支持”等表达，避免硬性催付感。
